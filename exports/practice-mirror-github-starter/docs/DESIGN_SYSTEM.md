# Practice Mirror - Design System

## Design target

Practice room, not therapy app and not corporate training software.

The app should feel focused, private, and serious. It should make rehearsal feel like a normal skill-building act.

## Layout

Primary screen:

- left panel: scenario goal, counterpart card, stakes, difficulty
- center: conversation transcript and input
- right panel: emotional temperature, success criteria, coach controls

Review screen:

- scorecard
- transcript highlights
- final script
- repeat drill

## Visual language

Use restrained contrast and clear state. Avoid cute wellness gradients. Avoid enterprise dashboard density.

Suggested palette:

- charcoal or ink base
- warm white transcript area
- muted red for pressure
- green for clarity / progress
- blue for coach mode
- amber for risk

## Components

- Mode selector
- Scenario setup form
- Counterpart role card
- Goal card
- Emotional temperature meter
- Transcript message
- Coach hint panel
- End session confirmation
- Review scorecard
- Final script editor

## Text rules

- Buttons should be direct: Start, Pause, Hint, End, Repeat, Save Script.
- Do not explain the whole product in the UI.
- Use compact labels inside panels.
- Keep AI feedback short during live practice.

## Interaction rules

- User can pause at any time.
- User can reset a scenario.
- User can delete a session.
- Coach mode is visually distinct from roleplay mode.
- The counterpart never appears as the coach.

## Demo requirement

The default route should open into the product experience. A first-time user should be able to start from a seed scenario without reading documentation.
