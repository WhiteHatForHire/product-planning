# Big Picture

NYC Sublet Radar is a practical consumer-agent experiment wrapped around a real need: finding a short-term NYC base without drowning in noisy housing listings.

The problem is not that listings are impossible to find. The problem is that the market is fragmented, time-sensitive, scam-heavy, and trust-mediated. A useful system does not need to defeat platforms. It needs to help a human operator see faster, choose better, and reply well.

## Core Thesis

Sublet Radar is a filter plus human closer.

The system handles:

- ingestion
- extraction
- deduplication
- scoring
- scam and mismatch flags
- shortlist assembly
- draft replies

The human handles:

- judgment
- final voice
- platform-native sending
- trust signal
- negotiation
- money
- identity

That split is the product.

## Why White Hat Is Better

For this problem, adversarial scraping is not clever. It is lower ROI.

White-hat sources give enough signal for a one-person search:

- Craigslist email alerts
- Listings Project
- Leasebreak
- Roomi
- SpareRoom
- Reddit RSS
- Substack/newsletter listings
- friend-of-friend Slack/Discord posts
- manually pasted Facebook listings

The trust signal matters more than automation. If a leaseholder is deciding whether to hand a stranger keys, the message should come from a real person on a real account.

## The Portfolio Value

If it works, Sublet Radar demonstrates:

- applied agent design on a hostile-information problem
- practical platform compliance
- human-in-the-loop automation
- personal CRM/memory for a real-world search
- fast product judgment under constraints

If it does not work, the architecture and build report still become a useful case study in disciplined scope control.

## Strategic Boundary

This is not a housing SaaS yet.

V0 exists to answer one question:

> Does AI triage materially improve Marcus's ability to find and reply to plausible NYC sublets?

Everything else waits.
