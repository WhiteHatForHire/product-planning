# Build Doctrine

## Phronesis Check

NYC Sublet Radar can be built in two registers:

- bad version: side quest dressed as need
- good version: bounded probe and portfolio artifact

The good version is lean, white-hat, and tied to a real housing search. It ships a useful digest or it stops.

## Hard Build Boundary

- Maximum V0 build budget: 5 focused sessions.
- Maximum spend before review: $100.
- No v0.5 until V0 helps produce at least one real reply.
- No work after 9pm local time.
- No automated outbound messaging.
- No adversarial scraping.
- Stop if setup exceeds five sessions without a working digest.

## V0 Success Criteria

All five are required before expanding scope:

- At least 30 real listings ingested from live sources.
- At least 10 listings normalized to structured JSON.
- At least 5 plausible shortlist items surfaced.
- At least 3 usable reply drafts produced.
- Marcus sends at least 1 real reply from the original platform using a system draft as source material.

## Kill Criteria

Any one stops the build:

- More than 5 focused sessions without a working digest.
- More than $100 spent without real listings flowing.
- No listings flowing after inbox/RSS setup.
- Build starts displacing higher-priority work.
- Build sessions creep late at night.
- Architecture expands into multi-user SaaS, payments, scraping, or platform automation.

## Architectural Principle

The V0 system has three modules:

1. Ingest + Normalize
2. Score + Flag
3. Digest + Draft

The six-agent architecture is a useful mental model for later. Do not build it first.

## Compliance Principle

Every ingestion adapter must declare:

```text
SOURCE:
METHOD:
COMPLIANCE:
TOS_NOTES:
```

Only `COMPLIANCE: white_hat` is allowed in V0.
