Symposium OS: The Cleaner Room Protocol

Version: 0.1
 Owner: Marcus Vale / Symposium Studios
 Purpose: A doctrine and operating protocol for making messy information environments agent-readable before asking AI to write, decide, code, summarize, advise, or execute.

1. Core thesis

AI does not just need better prompts. It needs a cleaner room.

In the early prompting era, humans were the organizers. We gathered the PDFs, meeting notes, decks, transcripts, spreadsheets, emails, screenshots, old drafts, and Slack threads, then tried to compress all of that into a prompt. That worked for small tasks, but it breaks down in serious work.

In the agentic era, the model can walk folders, read file trees, compare documents, summarize source sets, detect conflicts, and prepare its own working map.

That changes the primitive.

The real prompt is no longer just the instruction typed into the chat box.

The real prompt is the source environment.

If the folder is messy, the model reasons from mess. If files are stale, duplicated, badly named, contradictory, or missing context, the final output inherits those distortions. A brilliant model operating inside a dirty room can still produce polluted work.

The first job of serious AI work is not generation.

The first job is orientation.

2. The problem

Most serious projects do not begin with a clean source set.

They begin with a pile.

A company has old decks, new plans, outdated financial models, meeting transcripts, half-written strategy docs, support exports, buried emails, and contradictory naming conventions.

A writer has drafts, journal fragments, screenshots, voice transcripts, partial outlines, old titles, discarded versions, and notes that may or may not still matter.

A legal, consulting, or operational team has source documents scattered across Drive, Dropbox, Slack, email, Notion, local machines, shared folders, and personal desktops.

The human assumption is usually:

“The important stuff is probably in there somewhere.”

That assumption is not good enough for agentic work.

When an AI system is asked to generate a serious output from a messy source environment, the risk is not only that the model will hallucinate. The risk is that the model will faithfully synthesize the wrong room.

It may treat old numbers as current.

It may mistake duplicate drafts for corroboration.

It may elevate a stale proposal above the live operating plan.

It may miss the one transcript where the actual decision happened.

It may cite a source that exists but no longer governs.

It may produce polished language over an unverified working set.

That is how organizations get confident nonsense.

3. The cleaner room principle

The cleaner room principle is simple:

Before asking AI to produce serious work, ask AI to organize and audit the room it will reason from.

Do not start with:

“Write the memo.”

Start with:

“Inventory the source set.”

Do not start with:

“Analyze this company.”

Start with:

“Map the files, identify the source of truth, flag conflicts, and tell me what is missing.”

Do not start with:

“Rewrite my manuscript.”

Start with:

“Create a document inventory, identify draft lineage, separate canon from fragments, and produce a working brief.”

Generation comes after orientation.

Execution comes after inventory.

Authority comes after human review.

4. Why this matters now

Older AI workflows were mostly chat-based. The user fed the model a small bundle of context, then asked for output.

Newer AI workflows are agentic. The model can inspect a directory, read multiple files, traverse project folders, compare versions, create spreadsheets, write reports, open codebases, and act over time.

That makes file hygiene strategically important.

A file tree is no longer passive storage. It is cognitive infrastructure.

Bad file names are not just annoying. They are weak signals.

Duplicate drafts are not just clutter. They are reasoning contamination.

Unlabeled folders are not just messy. They are ambiguous prompts.

A stale PDF sitting next to a current operating plan is not harmless. It can distort the model’s understanding of reality.

In 2026, primitives matter again.

Naming matters.

Dates matter.

Metadata matters.

Version control matters.

Folder structure matters.

Canonical source designation matters.

Human approval gates matter.

The companies and creators who understand this will get more reliable leverage from AI. The ones who skip it will produce faster work with hidden fragility.

6. Core concepts

6.1 Source room

The source room is the total set of materials the AI is allowed or expected to reason from.

It may include:

documents

spreadsheets

PDFs

screenshots

transcripts

emails

decks

notes

code files

data exports

prior drafts

meeting notes

voice memos

decision logs

calendars

project boards

old proposals

folder structures

The source room is not automatically trustworthy. It must be inspected.

6.2 Working set

The working set is the subset of source material selected for a specific task.

A company may have 5,000 files, but a board memo may require only 30.

A memoir archive may have 10 years of material, but one chapter may require only one month.

A product rewrite may have many artifacts, but the current source of truth may be one spec, one repo, one backlog, and three decision logs.

The working set must be explicit.

6.3 Source of truth

The source of truth is the document, table, dataset, or decision record that governs a specific claim.

There may be multiple sources of truth by category:

current operating plan governs numbers

latest product spec governs feature behavior

signed contract governs obligations

transcript governs decision context

financial model governs forecasts

codebase governs implemented reality

user feedback doc governs product pain points

A source room without authority levels invites hallucination.

6.4 Stale source

A stale source is a document that may still contain useful context but no longer governs the current state.

Stale sources should not be deleted blindly. They often preserve history.

But they must be labeled.

The model should know:

current

background

superseded

deprecated

draft

archive

conflicting

unknown

6.5 Reasoning contamination

Reasoning contamination occurs when the model treats bad, stale, duplicate, or ambiguous material as valid evidence.

Common contamination sources:

duplicate drafts

old decks

old pricing sheets

partial transcripts

contradictory notes

stale strategy memos

ambiguous screenshots

unlabeled exports

files with misleading names

missing dates

unclear authorship

mixed personal and business folders

The cleaner room protocol is designed to reduce reasoning contamination before generation begins.

7. The Cleaner Room Protocol

Phase 1: Room scan

The agent first scans the folder, archive, repo, or shared drive.

It does not write the final deliverable.

It does not make conclusions.

It creates a map.

The output is a high-level inventory of what exists.

Required fields:

file name

file path

file type

date created or modified, if available

apparent date of content, if different

short description

likely category

apparent relevance

confidence level

notes or warnings

The goal is to answer:

“What room are we standing in?”

Phase 2: Source inventory

The agent creates a structured source inventory.

This is usually a spreadsheet or table.

Columns may include:

ID

file name

path

format

content type

date

author or source

project area

summary

key claims

key numbers

people mentioned

decision relevance

status

authority level

duplicate risk

conflict risk

recommended use

Status options:

canon

current

likely current

useful background

stale

superseded

duplicate

unclear

needs human review

exclude from working set

This becomes the first serious artifact.

Phase 3: Authority map

The agent identifies which documents appear to govern which domains.

Example:

“2026 Operating Plan.xlsx” governs current revenue projections.

“Board Deck Q1.pdf” is historical background only.

“Customer Interviews April Transcript.docx” governs user pain points.

“Product Spec v4.md” governs current product behavior.

“Product Spec v2.docx” is superseded.

“Slack Export May 3.txt” contains the actual decision on pricing, but needs human confirmation.

The authority map prevents the model from flattening all documents into equal evidence.

Phase 4: Conflict log

The agent compares important sources and identifies contradictions.

Conflict categories:

numbers differ

dates differ

names differ

goals differ

feature behavior differs

decisions differ

ownership differs

pricing differs

legal or contractual terms differ

strategic direction differs

Each conflict should include:

conflict ID

topic

source A

source B

contradiction

likely explanation

risk level

recommended human decision

The point is not for the AI to resolve everything automatically.

The point is to surface hidden ambiguity before the final output inherits it.

Phase 5: Duplicate and draft-lineage report

The agent identifies duplicates, near-duplicates, and draft families.

This matters because duplicates are not neutral. They can create false confidence.

If the same outdated claim appears in four duplicated drafts, the model may treat it as corroborated evidence.

The duplicate report should identify:

exact duplicates

near duplicates

older versions

renamed versions

draft lineage

likely latest version

files safe to archive

files requiring human review

For writing projects, this becomes especially important.

A memoir, essay collection, Substack archive, product manifesto, or book manuscript may contain multiple versions of the same thought. The system must distinguish development history from current canon.

Phase 6: Missing-source report

The agent asks:

“What seems to be missing?”

Examples:

The operating plan references a KPI export that is not present.

The draft memo references a meeting transcript that is missing.

The product spec mentions user feedback not found in the folder.

The financial model uses assumptions not documented anywhere.

The legal memo references a citation not present in the source set.

The memoir chapter refers to a date range with no journal entries included.

The codebase references environment variables not documented in the README.

This report is essential because models often proceed confidently without noticing absence.

A good agent should not only summarize what is present. It should identify what is missing.

Phase 7: Working brief

The working brief is the human-reviewed operating map for the task.

It tells the model:

what the task is

which sources are authoritative

which sources are background

which sources are stale

which conflicts remain unresolved

which assumptions are allowed

which claims require citation

which files should be excluded

what the final deliverable should and should not do

The working brief is the bridge between room cleaning and serious output.

Only after the working brief is approved should the AI begin the final work.

8. Standard artifacts

A complete Cleaner Room run produces some or all of the following artifacts.

8.1 Source Inventory

A spreadsheet or table listing every relevant file with summary, status, and recommended use.

8.2 Authority Map

A document identifying which sources govern which categories of truth.

8.3 Conflict Log

A structured list of contradictions, unresolved issues, and human decision points.

8.4 Duplicate Report

A map of duplicate, near-duplicate, outdated, and superseded files.

8.5 Missing Source Report

A list of referenced or implied materials that are not present in the working set.

8.6 Working Brief

A human-approved source map used to guide final generation or execution.

8.7 Final Output

Only after the room is cleaned does the system write the memo, build the product, draft the chapter, generate the report, or execute the strategy.

13. Prompt template: source room inventory

You are acting as a source-room auditor for a serious AI-assisted project.

Your first job is not to write the final deliverable.

Your first job is to inspect the source environment and make it legible.

Scan the provided folder or file set and produce a Source Inventory.

For each file, include:

- file name

- file path

- file type

- apparent date of content

- modified date if available

- short description

- likely category

- key topics

- key claims or numbers

- apparent status: canon, current, background, stale, duplicate, superseded, unclear, needs review, or exclude

- authority level: high, medium, low, unknown

- relevance to the stated project

- risks or notes

Do not produce the final memo, draft, report, or strategy yet.

After the inventory, provide:

1. Recommended working set

2. Files that appear stale or superseded

3. Files that may be duplicates

4. Files that require human review

5. Missing sources that seem necessary

6. Questions for the human operator before final generation

14. Prompt template: authority map

Using the Source Inventory, create an Authority Map.

Identify which documents appear to govern which categories of truth.

Categories may include:

- current strategy

- historical background

- financial numbers

- legal obligations

- product behavior

- customer feedback

- decision context

- implementation reality

- dates and timelines

- people and ownership

- open questions

For each category, identify:

- primary source of truth

- secondary supporting sources

- stale or superseded sources

- conflicts

- confidence level

- human review needed

Do not resolve uncertain authority by guessing.

Flag uncertainty clearly.

15. Prompt template: conflict log

Using the approved working set, create a Conflict Log.

Look for contradictions across documents.

Include conflicts involving:

- numbers

- dates

- names

- decisions

- ownership

- strategy

- product behavior

- legal terms

- customer claims

- timelines

- assumptions

For each conflict, include:

- conflict ID

- topic

- source A

- source B

- exact nature of the conflict

- likely explanation if inferable

- risk level

- recommended human decision

- whether final output should avoid this claim until resolved

Do not hide conflicts to make the source set look cleaner.

The purpose is to surface ambiguity before generation.

16. Prompt template: duplicate and staleness report

Review the source set for duplicate, near-duplicate, stale, or superseded documents.

Produce a Duplicate and Staleness Report.

For each cluster, include:

- related files

- likely draft lineage

- newest version

- most authoritative version, if inferable

- files that should be archived

- files that should remain as background

- files that should be excluded from final generation

- human review needed

Pay special attention to cases where old claims appear in multiple duplicated drafts.

Do not treat repeated claims as more reliable unless they come from independent current sources.

17. Prompt template: working brief

Create a Working Brief for the final AI-assisted task.

The brief should include:

- project goal

- intended final output

- approved working set

- source of truth by category

- background-only sources

- excluded sources

- unresolved conflicts

- missing materials

- assumptions allowed

- assumptions not allowed

- citation requirements

- tone and style requirements

- human decisions needed before final generation

The final output should not begin until the human operator approves this Working Brief.

18. Operating rules

Rule 1: No final output before inventory

Do not ask the agent to write, advise, or execute until it has mapped the room.

Rule 2: Authority must be explicit

The model must know which sources govern reality.

Rule 3: Duplicates are reasoning risks

Repeated stale claims can create false confidence.

Rule 4: Stale does not mean useless

Old documents may preserve context, but they should not govern current claims.

Rule 5: Missing sources matter

Absence should be surfaced, not silently ignored.

Rule 6: Human approval remains central

The agent can organize, flag, and recommend. The human operator approves the working brief.

Rule 7: Serious outputs require traceability

Claims should map back to sources.

Rule 8: The room is part of the prompt

Every folder, file name, and version label shapes the model’s reasoning.
