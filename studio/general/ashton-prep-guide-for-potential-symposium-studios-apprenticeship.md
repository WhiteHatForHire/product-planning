Ashton Prep Guide for Potential Symposium Studios Apprenticeship

Ashton, if you’re serious about potentially joining what I’m building with Symposium Studios, the main thing right now is not proving you’re a genius. It’s proving you can become organized, visible, consistent, and useful inside an AI-native build workflow.

1. Get your basics handled

First gate:

Get your passport process started.

Create a GitHub account if you don’t already have one.

Start backing up every project you care about to GitHub.

Make sure your computer/work setup is usable for focused build sessions.

Keep a simple folder or notes system for everything you are learning.

This field is moving fast. If you want to be considered seriously, you need to be able to move, document, and share your work.

2. Turn your VR game into proof

Since you said you’re making a VR game, treat that as your first proof artifact.

Do this:

Put the project on GitHub.

Write a simple README explaining what the game is.

Include screenshots or a short screen recording if possible.

List what works, what is unfinished, and what you want help with.

Track your progress in commits, not just in your head.

The goal is not perfection. The goal is evidence that you can build, learn, and document.

3. Learn GitHub and GitHub CLI

Minimum skills to learn:

create a repo

clone a repo

commit changes

push changes

pull latest changes

create branches

write decent commit messages

open issues

use GitHub CLI from the terminal

Use Claude Code or ChatGPT to tutor you through it. Don’t just watch videos. Actually do it.

4. Learn the AI-native workflow

Start practicing this loop:

Explain the goal clearly.

Ask AI to help create a plan.

Build one small piece.

Test it.

Commit it.

Document what changed.

Repeat.

Do not ask AI to build huge vague things. Learn to break projects into small, testable chunks.

5. Send me useful updates

When you update me, don’t just say “I’m working on it.”

Send:

GitHub link

what changed

what works

what is broken

what you learned

what you are doing next

That’s how I’ll know you’re actually moving.

6. What I’m looking for

I’m looking for:

follow-through

curiosity

honesty

documentation

willingness to learn

ability to take direction

ability to use AI without getting lazy

ability to show receipts

This is not about hype. It’s about becoming useful in a real build environment.

7. First assignment

Before anything else, do this:

Make or clean up your GitHub.

Put your VR game or another real project there.

Write a README.

Send me the link.

Start learning GitHub CLI with Claude Code.

Once I can see your work, we’ll have something real to talk about.
