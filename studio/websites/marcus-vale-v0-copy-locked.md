## Concept Index section

Section heading:

```
Concepts
```

Section intro (optional, one line):

```
A working vocabulary for the AI-native transition.
```

Concept entries (display as name in bold, definition in plain text on the same line or next line):

```
Director Model: The practice of directing AI agents to build software, rather than writing the code directly.
```

```
Council of Models: A structured multi-model review pattern. More than one AI in the room for high-stakes judgment.
```

```
Pattern Intimacy: Working closely enough with a system that its patterns become felt rather than analyzed.
```

```
AI Self-Governance: Using AI as a regulatory tool, not only a productivity tool. Structure, accountability, and attention.
```

```
The Minimum Viable Organization: The smallest unit capable of producing serious output. The unit is changing.
```

```
Four Modes: A framework for the seasons of a life. Exploration, Integration, Build, and Arena.
```

```
Practical Wisdom: Aristotle's phronesis. The judgment to act well in specific situations, where rules and principles give incomplete guidance.
```

```
Build Logs as Proof: The discipline of documenting how things were built. Credibility through receipts, not marketing claims.
```

```
AI as External Working Memory: Treating AI not as an oracle but as a memory and reasoning surface that holds what a single mind cannot.
```
