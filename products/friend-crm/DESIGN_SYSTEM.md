# Friend CRM - Design System

## Design target

Private intelligence desk, not enterprise CRM.

The interface should feel organized, slightly secretive, and fast to scan. It should avoid cartoon spy tropes and avoid corporate sales software tropes.

## Palette

Use a restrained neutral base with two accent families:

- Ink: near-black text and panels.
- Paper: off-white or pale gray surfaces.
- Signal green: active/healthy relationships.
- Warning red or amber: overdue, fragile, or risky items.
- Electric blue: selected/planning state.

Avoid a one-note dark blue or beige system.

## Typography

- Small dense headings.
- Readable body copy.
- No huge hero text inside the app.
- No marketing-style landing screen as the default route.

## Layout

Primary app shell:

- left sidebar for People, Radar, Plot Board, Reflection Log, Settings
- main workspace
- right detail rail for current person or selected next move

Important surfaces:

- compact table/list for people
- timeline for person detail
- board for social plans
- brief panel for pre-meeting context

## UI language

Use labels like:

- People
- Radar
- Plot Board
- Open Loops
- Next Move
- Warmth
- Context
- Boundaries
- People To Protect
- Drift

Avoid:

- Leads
- Prospects
- Pipeline
- Conversion
- Deal
- Campaign

## Components

- Person row
- Warmth chip
- Importance indicator
- Open loop card
- Memory card
- Next move card
- Sensitive note badge
- Pre-meeting brief panel
- Reflection composer
- AI extraction review checklist

## Empty states

Empty states should ask for action, not explain the whole product.

Examples:

- "Add the first person."
- "Capture what happened."
- "No open loops."
- "Nothing overdue."

## Interaction rules

- AI suggestions always require explicit save.
- Sensitive data should be marked near the content, not hidden in settings.
- The user should be able to dismiss suggested next moves without explaining why.
- Editing must be fast. Relationship memory is only useful if capture is low-friction.
