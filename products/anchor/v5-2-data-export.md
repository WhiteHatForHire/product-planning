### Tables queried

stable_profile        (from user_memory — verify actual table/column name in Phase A)

Phone numbers: excluded from all exports by default. Opt-in via toggle. When opt-in is off, strip phone number fields from every table row before composing.

### JSON export envelope (verbatim)

{

"exported_at": "...",

"format_version": "v1",

"user_id": "...",

"tables": {

"stable_profile": [],

"check_ins": [],

"commitments": [],

"practice_progress": [],

"practice_choices": [],

"practice_scenario_feedback": [],

"event_log": [],

"saved_syntheses": []

}

}

Field names stay close to DB/API reality. Do not over-normalize.

AI-generated content labeling in JSON:

- Add "generated_by": "Anchor AI" and "saved_by_user": true to saved_syntheses rows that are AI-generated and user-saved.

### CSV ZIP structure (verbatim)

anchor-export-YYYY-MM-DD.zip

profile.csv

check_ins.csv

commitments.csv

practice_progress.csv

practice_choices.csv

practice_scenario_feedback.csv

event_log.csv

saved_syntheses.csv

ZIP library: use `archiver` (acceptable). One CSV file per table.

AI-generated content labeling in CSV: add columns `generated_by` (value: Anchor AI) and `saved_by_user` (value: true) to saved_syntheses rows that qualify.

### Markdown export structure

Tone: "Here is a clear overview of the recovery work this person has recorded in Anchor." Not clinical, not therapeutic.

Sections in order:

1. Export overview — export date, user identifier, date range covered, note: "This export was generated from Anchor data."

2. Recovery focus — substance/behavior focus, start date, current day count, recovery approach if available

3. Commitments — completed, active, repeated themes — no over-interpretation

4. Check-in trends — averages and simple counts only, no giant raw dump

5. Practice work — modules completed, repeated practices, last practiced dates

6. Notable history — selected event_log entries: relapse/slip support events, milestone events, high-signal saved synthesis paragraphs (recency + event type weighted, top 10 maximum)

7. AI-generated saved reflections — only paragraphs the user opted to save, clearly labeled per line:

> Generated by Anchor AI, saved by user on [date]

8. Footer (verbatim):

This is not a medical record or clinical assessment. Anchor may summarize patterns, but does not predict relapse.

### File naming

anchor-export-YYYY-MM-DD.json

anchor-export-YYYY-MM-DD.md

anchor-export-YYYY-MM-DD.zip

Browser handles duplicate same-day filenames.

### Loading state copy (verbatim)

- Standard: "Preparing your export. This may take a moment."

- Large history (1000+ event_log entries): "This account has a larger history, so export may take a little longer."

Generate synchronously. If agent identifies a genuine timeout risk during implementation, document in BUILD_REPORT as deferred streaming path and implement synchronously for V5.2.

### Event logging

New event_log type string: export_generated

Summary strings (verbatim, no user content, no filenames, no phone numbers):

- "User exported data in JSON format."

- "User exported data in Markdown format."

- "User exported data in CSV ZIP format."

### Phone number toggle (verbatim)

- Label: Include phone numbers in export

- Default: off

- Helper copy: Phone numbers are excluded by default. Turn this on only if you want saved contact numbers included in your export.

### Settings UI

Location: Settings → Data (new section) → Export subsection

Data section structure:

1. Export your data — export button, format selector (JSON / Markdown / CSV ZIP), phone number toggle

2. Delete account — separate subsection, not adjacent to export button

Delete account: surface the existing deletion path. No new deletion logic.

Do not place Delete next to Export. Delete feels available but must not be accidentally clickable.
